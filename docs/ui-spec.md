# UI specification — MVP

## 디자인 원칙

- 다국어 라벨(한국어/영어 전환), 선택형 위주, 1차 색: 중립 그레이 + 단일 액센트(예: teal 600).
- 터치 영역 최소 44pt, 카드형 목록.

## 화면별 UI

### 스플래시 / 진입 (`index`)

- 로딩 인디케이터 → 인증·온보딩·탭으로 리다이렉트.

### 로그인·회원가입 `(auth)`

- 이메일, 비밀번호, 제출 버튼, 하단 링크(로그인↔가입).

### 온보딩 `(onboarding)/profile`

- 닉네임 (필수)
- 주 활동 지역 (텍스트 + “서울 기본 좌표” 버튼으로 lat/lng 프리필)
- 추위/더위 민감도: 세그먼트 또는 라디오 3단계
- 주 이동 수단: 칩 선택
- 출근/등교: 토글
- 알림 수신 동의: 스위치
- 완료 시 `onboarding_completed = true`

### 홈 `(tabs)/home`

- **상단 카드**: 지역명, 현재/체감 온도, 최고·최저, 습도, 풍속, 강수 문구, 일교차 한 줄
- **추천**: 카드 1~3 — 착장 요약(상의/하의/아우터), 한 줄 이유, 경고 배지(저만족 이력 시)
- **주의 포인트**: 텍스트 1줄 (바람/실내 등 휴리스틱)
- **Primary CTA**: “오늘 착장 기록”
- **Secondary**: “비슷한 날 보기”
- **제외**: 즐겨찾기 착장 바로가기 버튼은 홈에서 노출하지 않음 (기록/상세에서 접근).

### 착장 기록 `outfit/new`

- 사진: 선택 시 미리보기, 제거 가능
- 카테고리: 상의·하의·아우터·신발 — 각각 Picker 또는 칩 그리드 (문자열 enum 목록 상수)
- 액세서리·두께감: 다중 칩
- 상황 태그: 다중 칩 (PRD 6.4 예시)
- 이동수단, 활동량, 실내외 비중: 단일 선택
- 메모: 멀티라인 선택
- 저장: 유효성 실패 시 스낵바

### 착장 상세 `outfit/[id]`

- 썸네일, 카테고리 요약, 날짜, 날씨 한 줄
- 버튼: “감상 기록 (출발/중간/귀가)”, “만족도 입력”, “즐겨찾기”
- 만족도 폼: 5개 슬라이더 또는 스텝퍼 1~5, 재착용 의향, 개선점 칩

### 감상 `feeling/[outfitId]`

- 단계 표시: 1/3, 2/3, 3/3
- 체감 단일 선택 (PRD 6.5 예시 라벨)
- 불편 요인 다중 칩
- 메모
- 다음 / 완료

### 히스토리 `(tabs)/history`

- 날짜 헤더 + 카드: 썸네일, 만족도 별, 상황 태그 축약
- 탭 또는 필터: 상황 태그

### 비슷한 날 `similar/index`

- 오늘 날씨 요약 vs 과거 비교 한 줄
- 리스트: 유사도 % 또는 별점, 착장 요약, 만족도, 개선점 태그
- 정렬: 만족도순 / 유사도순 / 최신순 (세그먼트)

### 설정 `(tabs)/settings`

- 프로필 요약, 알림 권한 상태, 로그아웃
- 언어 선택: 한국어 / 영어

## 접근성

- 버튼에 `accessibilityLabel` (한글).

## 빈 상태

- 추천 없음: “기록을 쌓으면 추천이 나타나요”
- 히스토리 없음: 홈으로 CTA

## 관리자 콘솔 (웹)

- **로그인**: 이메일·비밀번호, 에러 메시지 명확히
- **대시보드**: 카드형 KPI — 사용자 수, 착장 기록 수, (선택) 최근 7일 착장 수, 열린 문의 수
- **사용자 목록**: `profiles` — 닉네임, 지역, 가입일, `is_admin`, **`account_disabled` 배지**, 행 클릭 → 상세
- **사용자 상세**: 프로필 요약, user id, 관리자 여부, **비활성 여부**, “편집” 버튼, 해당 사용자 최근 착장 링크(선택)
- **사용자 편집**: 닉네임, `default_region`, 냉/더 민감도, `onboarding_completed`, `account_disabled`(토글·경고 문구), 저장/취소
- **착장 목록**: 최근 N건 — 착용일, 요약, 만족도, user_id, **상세 링크**
- **착장 상세**: 카테고리·메모·날씨 연결 요약, `context_logs` / `rating_logs` / `feedback_logs` 요약 테이블 또는 카드
- **문의 목록**: 제목, 사용자, 상태, 생성일, 상세 링크
- **문의 상세**: 본문, 상태 선택(`open`~`closed`), 관리자 답변(멀티라인), 저장
- **공지 목록**: 제목, 활성, 기간, 수정 링크
- **공지 편집/작성**: 제목, 본문, `is_active`, 시작·종료 시각, 정렬 순서
- **비관리자**: “권한 없음” 전용 화면, 로그아웃 링크
- **스타일**: 데스크톱 우선, 테이블 + 사이드바 네비(단일 액센트 유지)
- **언어 전환**: 사이드바에서 한국어/영어 전환 가능
- **상태**: 로딩/에러/빈 목록 문구 명시

### 모바일 (연동)

- **설정**: “운영진에게 문의” 진입 → 제목·내용·전송, 성공/실패 피드백
- **비활성 계정**: “이용이 제한된 계정입니다” 류 안내 후 로그아웃
