# Outfit 로깅·추천 앱 — PRD 요약

## 제품 한 줄

날씨·상황·3단계 감정을 함께 기록하고, 과거 유사 일·규칙 기반으로 코디를 추천하는 **개인용** 모바일 앱.

---

## MVP 범위 (P0)

| 영역 | 포함 내용 |
|------|-----------|
| 인증 | 이메일/소셜 등 Supabase Auth 기반 로그인·회원가입 |
| 날씨 | Open-Meteo로 위치(또는 고정 좌표)·일별 요약 연동, `weather_logs`에 스냅샷 저장 |
| 코디 기록 | 사진(Storage)·태그·메모, 기록 시점 날씨·상황과 연결 |
| 상황(Context) | 태그/카테고리(예: 출근, 데이트, 운동) — `context_logs` 또는 outfit에 embedded |
| 감정 3단계 | 기록 플로우 내 3-phase feelings (예: 아침/점심/저녁 또는 착용 전/착용 중/하루 후) — `feedback_logs` 또는 outfit 하위 구조 |
| 평가 | 코디·하루 만족도 등 정량/정성 — `rating_logs` |
| 히스토리 | 날짜·필터 기반 목록·상세 |
| 유사 일 검색 | 날씨·계절·태그 유사도 기반 “비슷한 날” 목록 (클라이언트 또는 Edge Function) |
| 추천 | **규칙 기반** 점수화 (날씨 구간, 선호 태그, 과거 만족도 가중) — MVP는 휴리스틱 |
| 즐겨찾기 | `favorite_outfits`로 코디 북마크 |
| 푸시 | 로컬 알림 또는 FCM 경로 중 하나로 “기록 리마인더” 수준 (구현 깊이는 첫 배포에서 최소화 가능) |

## 데이터베이스 테이블 목록 (개념)

| 테이블 | 역할 |
|--------|------|
| `users` | Supabase `auth.users` 확장 프로필 (표시명, 타임존, 단위 설정 등) — `public.profiles` 패턴 권장 |
| `weather_logs` | 사용자·시각·위치·Open-Meteo 스냅샷(JSON 또는 정규화 컬럼): 온도, 강수, 바람, 코드 등 |
| `outfit_logs` | 코디 본문: 사진 경로(Storage key), 메모, 착용일, `weather_log_id` FK, 생성 시각 |
| `context_logs` | 상황 태그 이벤트: `outfit_id` 또는 일 단위 키, 태그 배열/enum |
| `feedback_logs` | 3-phase feelings: `outfit_id`, phase(1–3), 값(이모지/점수/라벨), 시각 |
| `rating_logs` | 만족도: `outfit_id` 또는 일 단위, 차원(전체/편안함 등), 점수 |
| `favorite_outfits` | `user_id`, `outfit_id`, 생성 시각 (유니크 제약) |
| `recommendation_logs` | 추천 요청·응답 요약: 입력 특징, 상위 후보 outfit id, 점수, 생성 시각 (디버깅·개선) |

관계: 한 사용자는 여러 `outfit_logs`; 각 outfit은 0~1 `weather_logs` 참조; `feedback_logs`·`rating_logs`는 outfit에 종속; `recommendation_logs`는 사용자 단위 감사 로그.

## 추천 공식 요약 (Rule-based)

입력: **오늘(또는 선택일)의 `weather_logs` 요약**, 사용자 **선호/자주 쓰는 context 태그**, **과거 `rating_logs`·`feedback_logs` 가중 평균**, **favorite** 가산.

1. **날씨 적합도**: 온도 구간(예: 매우 추움~더움)과 과거에 같은 구간에서 높은 점수를 받은 outfit 패턴 매칭.
2. **상황 일치**: 오늘 선택한 context와 과거 co-occurrence가 높은 outfit 가산.
3. **감정·평가**: 최근 N일 내 긍정 피드백이 많은 outfit 가중; 동일 아이템 스타일(태그) 반복 시 소폭 페널티(다양성, 선택적).
4. **최종 점수**: 가중 합 또는 곱셈 정규화 후 상위 K개; 동점 시 최근 착용일·다양성 타이브레이크.

MVP에서는 **설명 가능한 규칙**만 사용하고, 모델 학습은 하지 않음.

## MVP slice for first delivery (첫 배포 슬라이스)

다음 조합으로 **첫 릴리스 범위를 명시적으로 고정**한다.

- **클라이언트**: 모바일 앱 — **Expo**(Expo Router), 단일 사용자 플로우 중심.
- **백엔드**: **Supabase** — Auth + Postgres + Storage(코디 사진).
- **날씨**: **Open-Meteo** API (키 없이 가능한 공개 엔드포인트; 사용량·캐시 정책은 클라이언트/Edge에서 결정).
- **추천 실행 위치**: **규칙 기반 추천**을 **클라이언트에서 계산**하거나, 필요 시 **Supabase Edge Function**에서 동일 로직 실행 후 결과만 반환 (RLS·비밀 관리 용이 시 Edge 선호).

첫 배포에서는 **유사 일 검색 + 규칙 추천 + 기록·히스토리**가 동작하는 것을 완료 기준으로 하고, 푸시·고급 오프라인은 최소 기능 또는 P0 하한선만 적용한다.

## 원본 PRD와의 대응 (기능명세 요약)

- **온보딩**: 이메일 로그인, 닉네임, 주 지역, `cold_sensitivity` / `heat_sensitivity`, `default_transport`, 출근·등교 여부, 알림 동의.
- **날씨**: 현재·체감·최고/최저·습도·풍속·강수·상태·일교차 안내; 위치 미허용 시 기본 지역.
- **착장 기록**: 사진 선택, 상의/하의/아우터/신발 카테고리, 액세서리·두께감, 상황 태그(다중), 이동·활동량·실내외, 메모; 상의·하의 중 최소 1개 필수.
- **감상 3단계**: `first` / `middle` / `last`, 체감·불편 태그·메모.
- **만족도**: 전체·온도·활동·상황적합·스타일(1~5), 재착용 의향, 개선점 태그.
- **비슷한 날**: 기온·체감·풍속·습도·강수·상황·활동·실내외 유사도; 정렬: 만족도 / 유사도 / 최신.
- **추천**: 유사 과거 상위 n → 만족도·재착용·개선점 반영 점수, 최대 3건, 실패 이력 경고 1건 이하.

## 관리자 콘솔 (운영·감사 MVP)

| 항목 | 내용 |
|------|------|
| 목적 | 가입자·착장 데이터 **읽기 중심** 조회, 간단 지표로 운영 상태 파악 |
| 형태 | 웹 앱 `admin/` (Vite + React), 동일 Supabase 프로젝트 **anon 키 + 세션** |
| 권한 | `profiles.is_admin = true` 인 계정만 UI 접근; RLS로 타 사용자 행 **SELECT** 허용 |
| MVP 화면 | 로그인, 대시보드(건수 KPI), 사용자 목록, 착장 목록(최근 N) |
| 제외 | 콘텐츠 삭제/신고 처리, 역할 세분화, 서비스 롤 키를 브라우저에 노출 |

관리자 지정은 **SQL Editor**에서 `profiles.is_admin`만 갱신 (일반 사용자는 API로 스스로 승격 불가).

## MVP 완료 정의 (게이트)

- 오늘 착장을 선택 위주로 **약 1분 내** 기록 가능.
- 날씨 자동 연동(좌표 또는 기본 지역) 및 당일 스냅샷 저장.
- 3단계 감상·만족도/개선점 저장.
- 비슷한 날 **3건 이상** 조회 가능(데이터가 있을 때).
- 추천 카드 **1~3개** 표시.
- **푸시**: 알림 권한 UX + 로컬 리마인더 1종(또는 스케줄 가능한 훅)으로 “기본 푸시” 경로 검증.
