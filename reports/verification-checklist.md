# 화면별 검증 체크리스트

**목적**: 모바일·관리자 주요 흐름을 빠짐없이 확인하고, [test-report.md](test-report.md)·[gate-checklist.md](../docs/gate-checklist.md) 갱신 시 근거로 사용합니다.

---

## 1. 누가 무엇을 할 수 있는지

### 1.1 에이전트(CI/터미널)가 실행할 수 있는 것

| 검증 | 명령 / 방법 | 비고 |
|------|-------------|------|
| 모바일 TypeScript | `cd mobile && npm run typecheck` | 네트워크·기기 불필요 |
| 관리자 프로덕션 빌드 | `cd admin && npm run build` | Vite 번들 |
| 관리자 ESLint | `cd admin && npm run lint` | 규칙에 따라 경고만 있을 수 있음 |
| E2E 스크립트(가입/로그인 API 스모크) | `cd mobile && npm run test:e2e-signup` 또는 `npm run test:e2e-signin` | `.env`에 Supabase 변수·`E2E_EMAIL`/`E2E_PASSWORD` 등 필요. **Supabase 가입/메일 rate limit**에 자주 막힘 |
| Expo 웹 export(선택) | `cd mobile && npx expo export --platform web` | 번들 생성 여부 확인용 |

**에이전트가 할 수 없는 것(원칙)**

- 실제 폰/에뮬에서 터치·권한(위치·카메라·알림)·화면 전환 체감
- 브라우저에서 관리자 UI 클릭·세션 쿠키·로그인 폼 제출
- Supabase 대시보드에서 Storage/RLS/마이그레이션 수동 조작
- 실제 푸시·이메일 수신 확인

### 1.2 사용자(본인)가 직접 해야 하는 것

- 위 표에 **수동**으로 분류된 모든 UI·기기·브라우저 검증
- `.env` / `VITE_*` 실제 값 설정 및 **개발 서버 재시작** 후 동작 확인
- 관리자 계정 `profiles.is_admin = true` 지정(SQL)
- 스토어 배포·실제 사용자 테스트

## 2. 모바일 (`mobile/`) — 화면별

각 항목은 **수동(기기/에뮬)** unless noted.

### 2.1 진입·환경

| 화면/경로 | 확인 내용 | 담당 |
|-----------|-----------|------|
| `index` (스플래시·분기) | 미설정 시 `/setup`, 로그인/온보딩/탭 분기 | 수동 |
| `setup` | `.env` 없을 때 안내 문구 가독성 | 수동 |

### 2.2 인증 `(auth)`

| 화면 | 확인 내용 | 담당 |
|------|-----------|------|
| 로그인 | 잘못된 비밀번호·빈 값 시 피드백 | 수동 |
| 로그인 | 성공 시 다음 화면 이동 | 수동 |
| 가입 | 인증 메일 사용 시 안내 알림·로그인 화면 이동 | 수동 |
| 가입 | 429 등 에러 시 한국어 안내(가능한 범위) | 수동 |

### 2.3 온보딩 `(onboarding)`

| 화면 | 확인 내용 | 담당 |
|------|-----------|------|
| 프로필 | 필수값·지역·토글 저장 후 홈 진입 | 수동 |

### 2.4 탭 `(tabs)`

| 화면 | 확인 내용 | 담당 |
|------|-----------|------|
| 홈 | 날씨 로딩/에러/빈 추천 문구 | 수동 |
| 홈 | 추천 카드 1~3·CTA 동작 | 수동 |
| 히스토리 | 목록·빈 상태·상세 이동 | 수동 |
| 설정 | 로그아웃, 프로필 표시 | 수동 |
| 설정 | 알림: **Expo Go Android**에서는 미지원 안내, **개발 빌드**에서 권한·로컬 예약 | 수동 |

### 2.5 스택 화면

| 화면 | 확인 내용 | 담당 |
|------|-----------|------|
| `outfit/new` | 카테고리 검증·저장·Storage 업로드(정책·용량) | 수동 |
| `outfit/[id]` | 상세·즐겨찾기·감상/만족도 진입 | 수동 |
| `feeling/[outfitId]` | 3단계 저장 | 수동 |
| `similar/index` | 위치 권한·목록·정렬 | 수동 |

### 2.6 모바일 자동 명령(반복 시)

- [ ] `cd mobile && npm run typecheck` — **에이전트**
- [ ] (선택) `npm run test:e2e-signin` — **에이전트** (환경·한도 조건부)
